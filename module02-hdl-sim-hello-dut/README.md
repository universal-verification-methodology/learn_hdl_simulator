## Outcomes

After this module you can explain and practice the ideas taught by **`hdl-sim-hello-dut`**, in the browser literacy lab and/or the public HDL Simulator.

## Two tracks (pick one or both)

### Track A — Public simulator (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and try the same workflow in the [HDL Simulator](https://universal-verification-methodology.github.io/systemverilog-simulator/).
2. Complete [CHECKLIST.md](CHECKLIST.md).
3. Optional self-check: `./scripts/module.sh 02 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/hdl-sim-hello-dut/index.html](http://127.0.0.1:8080/tools/hdl-sim-hello-dut/index.html)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/hdl-sim-hello-dut/](https://universal-verification-methodology.github.io/learning/tools/hdl-sim-hello-dut/)
3. Load the **starter example**, then work challenges.
4. Check off the Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs are literacy tools — they do not replace time in the public simulator IDE.
